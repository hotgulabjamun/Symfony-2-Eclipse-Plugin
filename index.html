Symfony

Symfony is a popular PHP Web Development framework and a set of standalone components available as PHP libraries. This eclipse plugin aims to provide support for framework specific idioms. It is built on top of the Dynamic Languages Toolkit and the PHP Development Tools and adds some useful features which hopefully save you some time like:

  • - Codeassist for elements specific to the Symfony framework, like services, routes and template paths, entities and translations.
  • - Annotation support
  • - Twig support

Twig

Twig is a templating language for PHP. Symfony has built in support for Twig, and so does the Symfony Eclipse Plugin.

Doctrine

Doctrine is a set of PHP libraries primarily focused on providing persistence services and related functionality. It comes bundled with the Symfony Standard-Edition as the default ORM. The Symfony Eclipse Plugin also provides Doctrine support.

Yaml

Yaml is a data serialization standard which is supported by Symfony. If you're using yaml, there's a separate eclipse plugin available. which adds syntax highlighting and validation for yaml files.

Prerequisites

Currently a development build of the PHP Development Tools (PDT) is required to run the Symfony plugin. This development build is available in the updatesite of this plugin (see below).

Installation via Updatesite

The plugin can be installed via the regular update site mechanism.

The update site is located at

http://pulse00.github.com/Symfony-2-Eclipse-Plugin/


After adding the updatesite to your eclipse installation, you will see 2 categories:

"PHP Development Tools" is the current development version of PDT. This is required at the moment, the Symfony Plugin will not work with the current stable PDT version.

The Symfony Feature provides all necessary tools to fully integrate Symfony support into PDT/Eclipse.

Updating

When updating the plugin, eclipse will contact all available updatesites, which also includes the built-in updatesite hosting the current stable PDT plugin. This plugin will be shown in the "Available Updates" dialog:

Do not install this version of PDT, as it will break the symfony plugin! The development versions of the PDT plugin do not contain dashes or alphanumeric characters, this way you can make sure during an update that you're updating PDT from the correct site.

Usage

To make your existing PHP projects Symfony aware, add the SymfonyNature by right-clicking on the project in the PHP explorer -> Symfony -> Add/Remove Symfony Nature. This will add the Symfony project nature and trigger the indexing mechanism of DLTK/PDT/Symfony (see the status bar at the bottom).

For new Symfony projects, you can simply use the "New Symfony Project" wizard which sets up the project correctly.

To ease naviation in your Symfony project, the plugin offers hyperlinks for Symfony elements. When ctrl+clicking on such elements, PDT will know where to go and open the corresponding resource.

For example ctrl+clicking on 'session' in the method call $this->get('session'); will open the declaring type of the session service.

Currently supported elements:

  • - Services: open the declaring PHP type of the service
  • - Routes: open the corresponding controller when ctrl+click on a route in a template
  • - Assets: open the corresponding css/js file
  • - Template variables: open the corresponding controller(s) when ctrl-clicked
  • - ViewPaths: open the template when ctrl+clicking on a viewpath ie.

    $this->render('MyBundle:Controller:index.html.twig')

If you have ideas for additional elements, please create an issue on github.

file_link_format

Symfony provides a configuration flag to render file links in exceptions using a specific format. Unfortunately eclipse does not register an URL protocol in the operating system, so this does not work out of the box for eclipse. There's however a workaround (currently only available on Mac OSX):

  • 1. Download the symfony2eclipse.zip file from the download section in the github repository.
  • 2. Unzip the symfony2eclipse app, and copy the file in your application folder.
  • 3. Add an eclipse link in your FrameworkBundle:

  • 4. Add the ide node to your configuration:

    • File links in Symfony exception pages should now open / focus eclipse and open the corresponding file.

    The source of the symfony2eclipse app is available in the github repo. It's a simple applescript registering a symfony:// URL scheme with OSX via the info.plist file of the applescript-bundle. When the browser requests that URL scheme, the script handles it and opens Eclipse.

    Any hints (or even better pull requests) how to implement this for Windows and Linux are greatly appreciated!

How do i know a PHP project is symfony-aware?

Check for a "Symfony Builder" in the projects builders - project -> Properties -> Builders. If it's not there, run the "Add/Remove Symfony Nature" (right-click on the project in the package explorer -> Symfony -> Add/Remove Symfony Nature).

Service XYZ is not being completed at all, but others are. How come?

The plugin cannot infer types of synthetic services - services which are dynamically injected during runtime. For those cases, there's a preference page called Services where the id/class mapping can be stored. After that, the plugin should complete the service properly.

There's no additional codeassistance showing up, what can i do?

The plugin indexes services and annotation during the build process. Try to clean the project (project -> clean). If it's still not working, check if the project has a Symfony nature.

Still no code assist, what now?

Check your build-path - project -> properties -> PHP build path - and ensure that all your source folders are in the build path. Also make sure that none of your sourcefolders are excluded.

The plugin broke the built in PHP code completion!?!

Sorry about that ;) To check that it's really the plugins fault, first remove the symfony nature, clean your project and try again. If you're getting code assist, it's probably the plugin causing it. If not, you're build-path is probably misconfigured.

Okay, it seems the plugin broke it, how can i figure out what's wrong?

Eclipse can be started with various debugging options. This can be turned on/off for each installed plugin independently. To debug the symfony plugin, create a file called .options in your Eclipse installation folder. The file should contain the following lines:

com.dubture.symfony.core/debug=true
com.dubture.symfony.index/debug=true
com.dubture.symfony.ui/debug=true

Then restart eclipse from the commandline with these flags: eclipse -debug .options

Try to reproduce the problem and monitor the commandline - please provide these stacktraces in your github issues.

How do i know which version of the plugin is installed?

Eclipse(Mac) / Help(Windows/Linux) -> About Eclipse -> SDK -> Installation Details -> Plugins lists all installed plugins and their version. Note that the Symfony eclipse Feature actually consists of multiple plugins listed on that page.
