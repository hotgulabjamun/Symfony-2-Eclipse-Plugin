Launch Symfony Controllers/Actions in XDebug debugger

The first step to debug your Symfony actions is to create a webserver in your eclipse PHP preferences.

You'll see an additional option in the "New Symfony project" wizard which lets you create new projects directly in your DocumentRoot:

To make XDebug work correctly, select the "XDebug (Symfony)" PHP Debugger in the project's options (this can be configured on a workspace scope as default for all your projects):

After that, create breakpoints somewhere in an action of your controller and select "Symfony action" from the "Debug as" menu:

Eclipse will switch to the debug view and set the correct URL of your action into the browser:

You can of course set the default browser which will be used for the launch to anything you prefer in the Eclipse preferences.
